# Código de Conducta

## Security Edge

El proyecto **Security Edge** busca mantener un entorno de trabajo profesional, respetuoso, colaborativo e inclusivo para todas las personas que participen en su desarrollo.

---

# Nuestro Compromiso

Nos comprometemos a promover un ambiente donde todas las personas puedan participar sin importar su experiencia, nivel técnico, edad, nacionalidad, identidad o cualquier otra condición personal.

Esperamos que todas las interacciones se desarrollen con respeto, responsabilidad y profesionalismo.

---

# Comportamientos Esperados

Se espera que todos los colaboradores:

- Actúen con respeto hacia los demás.
- Mantengan una comunicación clara y profesional.
- Acepten críticas constructivas.
- Documenten adecuadamente sus contribuciones.
- Colaboren para mejorar la calidad del proyecto.
- Respeten las decisiones técnicas adoptadas por el equipo.
- Reporten problemas de manera responsable.

---

# Comportamientos Inaceptables

No serán aceptados comportamientos como:

- Acoso o intimidación.
- Lenguaje ofensivo o discriminatorio.
- Amenazas o agresiones.
- Publicación de información privada sin autorización.
- Sabotaje deliberado del proyecto.
- Uso malicioso del código fuente.
- Difusión de vulnerabilidades sin seguir la política definida en `SECURITY.md`.

---

# Responsabilidades del Mantenedor

El mantenedor del proyecto podrá:

- Revisar contribuciones.
- Solicitar cambios.
- Rechazar Pull Requests que no cumplan los estándares.
- Eliminar contenido inapropiado.
- Restringir la participación de usuarios que incumplan este código.

Las decisiones deberán tomarse de manera objetiva y buscando siempre el beneficio del proyecto.

---

# Alcance

Este código aplica a:

- Repositorio de GitHub.
- Pull Requests.
- Issues.
- Documentación.
- Código fuente.
- Debates técnicos.
- Cualquier espacio oficial del proyecto.

---

# Incumplimiento

Las violaciones a este código podrán derivar en:

1. Advertencia.
2. Solicitud de corrección.
3. Restricción temporal de participación.
4. Expulsión definitiva del proyecto en casos graves.
